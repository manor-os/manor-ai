# Image Tools

Image tools cover system-assisted AI generation manifests and image inspection for the PPTX generation path.

## `image_prompts.py`

Manifest utility for AI-generated image rows. It never calls external image APIs.

The built-in generated-image flow is:

1. Write `project/images/image_prompts.json`.
2. Render/maintain `project/images/image_prompts.md` as the prompt audit sidecar.
3. Use the system AI image generation tool available to the agent.
4. In Manor, copy the returned workspace file into the project with `import_system_image.py`.
5. Mark the corresponding manifest item `Generated`.

```bash
python3 scripts/image_prompts.py --validate project/images/image_prompts.json
python3 scripts/image_prompts.py --render-md project/images/image_prompts.json
python3 scripts/image_prompts.py --mark project/images/image_prompts.json \
  --filename cover_bg.png --status Generated
python3 scripts/image_prompts.py --check-files project/images/image_prompts.json
```

## `import_system_image.py`

Copy an image generated by Manor's system `generate_file(kind="image")` tool
from the sandbox's read-only `/workspace` mount into `project/images/`.

Use the `fs_path` returned by `generate_file` when present:

```bash
python3 scripts/import_system_image.py \
  --workspace-path Workspaces/acme/images/cover_bg.png \
  --project project \
  --filename cover_bg.png
```

If the tool only returns `image_url`, pass it directly:

```bash
python3 scripts/import_system_image.py \
  --image-url /api/v1/fs/<entity_id>/Workspaces/acme/images/cover_bg.png \
  --project project \
  --filename cover_bg.png
```

## `analyze_images.py`

Analyze images in a project directory before writing the design spec or composing slide layouts.

```bash
python3 scripts/analyze_images.py <project_path>/images
```

Use this instead of opening image files directly when following the project workflow.
